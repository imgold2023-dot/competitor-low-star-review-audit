# Shopify 低星评论审计回复模板

日期：2026-06-04  
用途：Day 5-6 发布后，人工回复评论、询价和质疑。  
边界：这些是草稿，创始人发布前需要自行确认语气、承诺和联系方式。

## 有人问“能不能看我的类目”

英文：

```text
Yes. Send me the category and 3-5 public competitor links if you already have them.

I will first check whether the category has enough public evidence for this format. If the evidence is too thin, I will say no rather than force a weak report.
```

中文：

```text
可以。你发一个类目，最好再带 3-5 个公开竞品链接。

我会先判断这个类目公开证据够不够。如果评论、价格和替代方案都太少，我会直接说不适合做，不硬出报告。
```

## 有人问价格

英文：

```text
For the first validation week, I’m testing a standard audit at $49-$99.

The standard version covers 3-5 public competitors, low-star review buckets, substitutes, decision labels, and a 7-day validation suggestion. No account login or private data.
```

中文：

```text
第一周先测标准版 $49-$99。

标准版看 3-5 个公开竞品、低星评论分类、免费替代方案、Proceed / Narrow / Hold / Reject 判断，以及 7 天最小验证建议。不需要账号和私密数据。
```

## 有人说“我自己也能看评论”

英文：

```text
That’s fair. If the buyer already has time and a clear decision frame, they should do it themselves.

The paid version is for people who want a second pair of eyes before building: which complaints repeat across competitors, which ones are noise, and what not to build first.
```

中文：

```text
这个判断是对的。如果你有时间，也有清楚的判断框架，完全可以自己做。

付费版更适合想在开发前找一个外部视角的人：哪些抱怨跨竞品重复，哪些只是噪音，哪些方向不该先做。
```

## 有人要求登录后台或看订单

英文：

```text
I can’t take private account access, order data, customer data, cookies, payment credentials, or refund/dispute workflows.

This audit only uses public information. If the question requires private data, it is outside this offer.
```

中文：

```text
这个我不能接。这个审计只使用公开信息，不接收账号、订单、客户隐私、cookie、支付凭证，也不处理退款和争议流程。

如果问题必须依赖后台数据，就不属于这次服务范围。
```

## 有人问能不能做月度监控

英文：

```text
Not open yet. I’m first validating whether a single audit is useful enough.

If there are several requests for the same category, monthly monitoring could be a later version.
```

中文：

```text
暂时不开。现在先验证单次审计是否足够有用。

如果同一类目有多人询问，月度监控可以作为后续版本。
```
