# 竞品低星评论审计公开页中文伴随版本

日期：2026-06-04  
对应公开页：`index.html`  
用途：供内部审阅、中文渠道改写和后续迁移使用。

## 首屏

标题：

`在开发下一个功能前，先读竞品低星评论。`

正文：

你给一个产品类目，或者 3-5 个公开竞品链接。我会看公开价格、低星评论、社区讨论和免费替代方案，然后给你一份简短判断报告。

报告会回答：

- 哪些抱怨在多个竞品里重复出现。
- 哪些抱怨只是噪音。
- 哪些方向值得 7 天内验证。
- 哪些方向应该收窄、暂停或避开。

边界：

- 不需要客户账号、订单数据、支付数据、后台数据或 cookie。
- 只处理公开信息。
- 验证期只收询价，不自动收款。

## 样例

样例类目：Shopify 退货/换货应用。

样例覆盖：

- AfterShip Returns
- Loop Returns
- Return Prime
- ReturnGO
- Shopify 原生退货/换货能力

样例结论：

Shopify 退货/换货类目不是不能进，但第一步不应该做另一个完整退货平台。成熟竞品已经覆盖退货门户、标签、换货、规则、报表和 store credit。更值得看的，是低星评论里重复出现的具体摩擦。

## 样例中的机会判断

| 判断 | 切口 | 原因 |
| --- | --- | --- |
| Proceed | 退货应用上线前检查清单 | 商家常在配置后才发现套餐、标签或流程限制 |
| Proceed | 某类 App 的低星评论审计 | 当前样例本身就是可交付格式 |
| Narrow | 低退货量商家的轻量流程方案 | 有需求，但价格敏感 |
| Narrow | Shopify Markets / 多币种退换货审计 | 痛点更窄，但更像高价定制 |
| Hold | 新做完整退货平台 | 支持、集成和迁移成本重 |
| Reject | 全自动退款/退货处理 | 涉及资金、争议和客户承诺 |

## 付费审计包含什么

这不是大而全市场报告，而是开发前的判断材料。

标准审计包含：

- 3-5 个竞品公开快照。
- 公开低星评论分类。
- 免费替代方案和平台原生能力。
- Proceed / Narrow / Hold / Reject 判断。
- 7 天最小验证动作。
- 风险和边界说明。

## 适合谁

- 独立开发者。
- 小型 SaaS 团队。
- Shopify App 开发者。
- Chrome 插件开发者。
- n8n / Make / Zapier 模板或自动化服务商。

## 不适合谁

- 需要登录客户后台的人。
- 需要查看订单、退款、客户隐私或支付信息的人。
- 想自动处理退款、争议或法律承诺的人。
- 想得到“这个产品一定赚钱”结论的人。

## 询价 CTA

你可以发一个产品类目，或者 3-5 个公开竞品链接。我会先判断这个类目适不适合做低星评论审计。如果证据太薄、范围太泛或风险太高，我会直接说不适合。

## 公开边界

本服务只使用公开信息。

不会使用：

- 客户登录账号。
- 私有后台数据。
- 生产 token。
- cookie。
- 支付凭证。
- 订单、退款、争议或客户隐私。

收款、退款、税务、外汇、交付承诺和最终发布仍由创始人确认。
